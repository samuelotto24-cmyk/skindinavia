Animation Direction for Skindinavia

Bold kinetic motion + micro-interaction polish. No scroll-fade-ins. Layers + entrance animations.

Proposed Animation Stack

A

GSAP + Framer Motion

GSAP for the theatrical stuff — text splitting/staggering, magnetic cursors, morphing, timeline-based entrance sequences. Framer Motion for component-level spring physics, layout animations, and page transitions.

Pros

  • Industry standard for premium animation (Apple, Nike use GSAP)
  • Best text animation capability (SplitText plugin)
  • Spring physics from Framer Motion feel organic
  • Great React integration with both

Cons

  • Two animation libraries = larger bundle (~45kb combined)
  • GSAP SplitText is a paid plugin (but we can replicate core behavior)
B

GSAP Only (with ScrollTrigger)

All-in on GSAP. Timeline-based entrance animations, text effects, magnetic cursor, morphing — plus ScrollTrigger for pinning and layer effects (not fade-ins, but parallax layers and section reveals).

Pros

  • One library, one mental model
  • ScrollTrigger pinning enables the layered depth you want
  • Smaller total bundle (~25kb)

Cons

  • No spring physics (GSAP eases are different from spring)
  • Layout animations less natural than Framer Motion
C

Framer Motion + Custom CSS

Framer Motion for spring physics, layout animations, page transitions, and entrance animations. Custom CSS for text effects and cursor interactions. No GSAP.

Pros

  • Built for React — declarative, easy to maintain
  • Best spring physics in the ecosystem
  • AnimatePresence for page/component transitions

Cons

  • Weaker for complex timelines and text splitting
  • No magnetic cursor built-in (custom impl needed)
  • Can't match GSAP's theatrical control
